Layer 2 Discovery

Unlike ICMP pings, ARP operates at the Data Link layer, discovering devices even if they have firewalls configured in stealth mode or ignore ping requests completely.

Vendor OUI Lookup

Every MAC address contains a 3-byte Organizationally Unique Identifier. The tool maps these to manufacturers like Cisco, Dell, or Apple using IEEE registrations.

ARP-Scan Functions Explained

Local Network Scan (-l)

Scans all devices on the local network segment using automatic interface detection.

Interface-Specific (-I)

Target a specific network adapter when you have multiple NICs or virtual interfaces.

Custom Subnet

Scan any subnet in CIDR notation (e.g., 10.0.0.0/8) for targeted reconnaissance.

High-Accuracy Retry (-r)

Send multiple ARP requests per host to reduce false negatives from packet loss.

Bandwidth Control (-b)

Throttle packet transmission rate. Higher values mean faster scans, lower values are gentler on the network.

Duplicate Detection (-d)

Identify IP conflicts where multiple MAC addresses respond to the same IP address.

Stealth Mode (-R)

Randomizes source MAC to avoid correlation with your real hardware address during scanning.

Verbose Output (-v)

Display detailed packet information including timing, retries, and protocol-level details.

How Classification Works
  1. ARP Request Broadcast: The tool sends Who-has packets to all IPs in the scan range
  2. Hardware Response: Devices reply with their MAC addresses at Layer 2
  3. OUI Database Matching: The first 3 bytes of the MAC are matched against IEEE vendor registrations
  4. Trust Verification: IPs and MACs are checked against trusted_ips.txt to flag unauthorized devices
